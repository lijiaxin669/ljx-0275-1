## 1. 产品概述

郊野亲子营地全景导览系统——为家长提供 8K 沉浸式 360° 全景地图，精准定位洗手点、急救站等关键设施，消除平面图导航盲区。

- **核心痛点**：现有平面图信息密度低，家长无法快速找到洗手点与急救站
- **目标价值**：3 秒内可拖动交互，热点一键获取设施说明与步行时间，移动端支持陀螺仪环视

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 访客 | 无需注册 | 浏览全景、查看设施信息 |
| 管理员 | 后台授权 | 编辑热点、上传全景切片 |

### 2.2 功能模块

1. **全景首页**：8K 立方体面全景球面渲染、拖动旋转、缩放、LOD 按视角加载
2. **热点交互**：地面热点标记、点击弹出设施说明卡片与步行时间
3. **移动端陀螺仪**：DeviceOrientation 自动切换、不支持时隐藏提示
4. **切片服务**：Node 端立方体面切片 API、静态资源托管

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 全景首页 | 全景渲染器 | Three.js 球面全景，6 面立方体贴图，支持拖动/缩放/双指 |
| 全景首页 | LOD 引擎 | 按视角方向加载当前+相邻立方体面，首屏仅加载正面 3 面 |
| 全景首页 | 热点系统 | 地面热点 Sprite，点击弹出设施卡片（名称、图标、步行时间） |
| 全景首页 | 陀螺仪控制 | DeviceOrientationControls，不支持时隐藏提示 UI |
| 全景首页 | 加载进度 | 首屏 3 秒内可拖动，渐进式加载提示 |
| 管理后台 | 热点编辑 | CRUD 热点位置、设施信息（预留接口） |

## 3. 核心流程

用户进入营地官网 → 加载首屏低分辨率立方体面（正面 3 面） → 3 秒内可拖动 → 拖动/旋转环视营地 → 发现地面热点 → 点击热点 → 弹出设施说明卡片（名称、图标、步行时间） → 移动端倾斜手机即可环视（陀螺仪）

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：森林绿 #2D5A27 + 暖沙色 #F5E6D3
- **辅助色**：急救红 #E53935（急救站标记）、水蓝 #1E88E5（洗手点标记）
- **按钮风格**：圆角胶囊按钮，半透明磨砂玻璃效果
- **字体**：标题使用 ZCOOL XiaoWei（站酷小薇），正文使用 Noto Sans SC
- **布局**：全屏沉浸式全景，浮动 UI 层叠
- **图标**：Lucide 图标库，设施类型使用对应图标

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 全景首页 | 全景渲染器 | 全屏 Canvas、半透明顶栏、营地名称 Logo |
| 全景首页 | 热点标记 | 圆形脉冲动画 Sprite、图标+文字标签 |
| 全景首页 | 设施卡片 | 底部弹出 Sheet、设施名称、图标、步行时间、关闭按钮 |
| 全景首页 | 陀螺仪提示 | 右上角浮动图标、检测失败自动隐藏 |
| 全景首页 | 加载指示 | 环形进度条、百分比文字 |

### 4.3 响应式

- 桌面端：鼠标拖动旋转、滚轮缩放
- 移动端：触摸拖动、双指缩放、陀螺仪环视
- 热点卡片移动端从底部弹出，桌面端从热点位置弹出

### 4.4 3D 场景指引

- **环境**：8K 全景立方体贴图，郊野营地实景
- **光照**：环境光贴图提供，无需额外灯光
- **相机**：透视相机，FOV 75°，近裁面 0.1，远裁面 1000
- **交互**：OrbitControls（桌面）+ DeviceOrientationControls（移动）
- **热点**：Sprite 定位在球面内侧，始终朝向相机
- **LOD**：按视角加载当前面 + 相邻 2 面，远处面使用低分辨率

## 5. 切片命名规范

```
pano/
  {scene-id}/
    {face}_{lod}.jpg
    ├── px_0.jpg   # 正X面 LOD0 (低分辨率 512×512)
    ├── px_1.jpg   # 正X面 LOD1 (中分辨率 1024×1024)
    ├── px_2.jpg   # 正X面 LOD2 (高分辨率 2048×2048)
    ├── nx_0.jpg   # 负X面 LOD0
    ├── nx_1.jpg   # 负X面 LOD1
    ├── nx_2.jpg   # 负X面 LOD2
    ├── py_0.jpg   # 正Y面(顶面) LOD0
    ├── py_1.jpg
    ├── py_2.jpg
    ├── ny_0.jpg   # 负Y面(底面) LOD0
    ├── ny_1.jpg
    ├── ny_2.jpg
    ├── pz_0.jpg   # 正Z面 LOD0
    ├── pz_1.jpg
    ├── pz_2.jpg
    ├── nz_0.jpg   # 负Z面 LOD0
    ├── nz_1.jpg
    ├── nz_2.jpg
    └── meta.json  # 热点数据、场景元信息
```

面标识：`px`=正X(右)、`nx`=负X(左)、`py`=正Y(顶)、`ny`=负Y(底)、`pz`=正Z(前)、`nz`=负Z(后)
LOD 级别：`0`=512px、`1`=1024px、`2`=2048px（原始 8K 切片输出 4096px 为 LOD3 可选）

## 6. 性能预算表

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 首屏可交互时间 (TTI) | ≤ 3s | 仅加载 LOD0 正面3面 |
| 首屏资源体积 | ≤ 300KB | 3面×512px×JPEG≈75KB×3 + JS≈75KB |
| 全量资源体积 | ≤ 5MB | 6面×2048px×JPEG≈300KB×6 + 热点数据 |
| 单帧渲染时间 | ≤ 16ms | 60fps，仅渲染可见面 |
| 内存占用 | ≤ 150MB | 最多同时加载 3 面 LOD2 |
| JS Bundle 体积 | ≤ 200KB (gzip) | Three.js tree-shaking + 业务代码 |
| 热点响应时间 | ≤ 100ms | 点击到弹出卡片 |
| 移动端帧率 | ≥ 30fps | 中端手机，LOD1 |
